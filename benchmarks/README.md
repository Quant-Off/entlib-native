# 벤치마킹

이 네이티브 라이브러리의 벤치마킹은 `criterion` 크레이트를 통해 진행되며, 보안성(security) 및 처리량(throughput)을 평가합니다.

## 환경

벤치마킹이 수행된 물리적 환경은 다음과 같습니다. 제 환경은 거의 고정적입니다. 환경을 바꿀 여력이 안 되기 떄문입니다...

- macOS (Ventura 13.7.8(22H730))
- 2.8 GHz 쿼드 코어 Intel Core i7
- 16GB 2133 MHz LPDDR3

> [!TIP]
> `cargo criterion` 명령을 통해 벤치마크가 가능하며, 수행된 경우 `target/criterion/report` 에서 정적 프론트엔드 파일을 통해 확인 가능합니다.

벤치마킹 결과는 수행 시간을 베이스로 측정되기 떄문에 다음의 초 단위를 사용합니다.

- ms(밀리초, $`10^{-3}`$)
- μs(마이크로초, $`10^{-6}`$)
- ns(나노초, $`10^{-9}`$)
- ps(피코초, $`10^{-12}`$)

특정 평가, 예를 들어 처리량 평가에서는 예외적으로 `Melem/s` 이라는 단위가 등장할 수도 있습니다. 이는 "Million Elements per second"의 약어이며, **"초당 백만 개의 요소 처리량"** 을 의미합니다.

## 목록

체크박스가 활성화된 문서는 벤치마킹이 모두 완료되었으며, 수정 가능성만을 염두해두고 있음을 의미합니다. 그렇지 않은 문서는 벤치마킹 진행 중 또는 완료되었으나 아직 그 내용이 작성되지 않았음을 의미합니다.
